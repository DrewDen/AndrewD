Memory Game

This project is a complete browser-based card matching game (also known as Concentration). The game board consists of sixteen cards arranged on a grid. The deck is made up of eight different pairs of cards, with various symbols. Cards are arranged randomly face down. When a card is clicked, it remains face-up until the next card is clicked. The rules are simple: flip a card and match the symbols. If they match, the cards will stay flipped and turn green. If they don’t, they will both flip over returning face down.

Features are added to the game to enhance gameplay such as a timer and star rating. The timer starts as soon as the first card is flipped over. The timer will stop when all the cards in the game match. The stars give the player a rating of how well they did at the end of the game. When all the cards match, a modal will appear to congratulate the player. The modal will show how long it took, how many stars received and how many moves it took.

The memory game was the first project in which I used HTML, CSS and JavaScript together. I also used DOM manipulation, bowser events and a modal. The challenging parts of this project were adding the time clock and star rating. I based the star rating on how many turns a player took. A ‘turn’ happens after the second card is flipped. After so many turns a star is removed from the group. The challenge is to make fewer turns throughout the game.

Classic Arcade Clone

I built a classic arcade clone of the game “Frogger” with a little twist on the characters and enemies. Visual assets and a game loop engine came with the project. By using these tools, I added several entities to the game, including player characters, objects and enemies.

By using object-oriented programming, I created access and modify objects. By learning this programming, I could design classes capable of creating countless instances of similarly functioning objects.

I mainly used JavaScript throughout the project. I used CSS for a title and bottom-border image. An opportunity came to study and learn a little about HTML 5 Canvas and ES6 with this project. The challenging part was taking a deep dive into JavaScript functions and learning how they worked.

Restaurant Review App

The goal was to take a static webpage to a mobile-ready web application. The design lacked accessibility and was not responsive to different sized displays. It also lacked features for screen reader use. A service worker was needed to cache files for offline use.

This project used a map box API to show locations of restaurants in New York. This concept could be used for any situation to review restaurants. One of the requirements was not to use a CSS framework. I used Flexbox and CSS position throughout the project. A tab index and aria attributes were used to help with the screen reader issues.

The service worker was used to cache requests to certain assets of the site. A unique feature was that if the user visited a page it would be accessible offline. A challenge faced for this project was making the offline part available. If the service worker was not shown doing its job, then it would have been useless.

Feeder Reader Testing

Test-Driven Development, also known as (TDD), is an approach where a test is written before you write enough production code to fulfill that test and refactor. In this project, I wrote tests for an existing web-based application that reads RSS feeds. By writing useful tests, I analyzed certain aspects of the JavaScript code. This is a great skill to have when changing teams or joining a new company.

Using the Jasmine testing framework, I wrote tests against a pre-existing application. The test checks the logic of the application, along with the event handling and DOM manipulation. A challenge I faced was knowing what a test suite is and how to use it. To understand this better, an experienced developer, who was a friend, did a small exercise with me. Our task was to write tests for a JavaScript calculator. This helped give me a better grasp of the concept.

Donate & Find Prototype

A prototype developed and designed for the Chingu Voyage Nine Program. My team and I decide to take the Front-End role for a project called Donate n Find. The idea was to provide a no-hassle process for donating and finding items, in which the community could see the process uncomplicated. The ideas we had for the project came from scratch and were not cloned from a website.

The Voyage Project was six week-long sprints. During that time, you design, develop and deploy the team's project. An MVP (Minimum Viable Product) was essential to establish early. MVP is defined as the minimum number of features to give the user a solid understanding of the project.

The Tech Stack decided upon HTML, CSS, JQuery, Bootstrap and SASS. Chingu wanted use of Git and GitHub for version control, with storing the main repo on GitHub. Teams decided on a developer workflow for things like, pull requests, merges, merge conflicts and project assignments. We integrated tools like Git and ZenHub to streamline our work along with learning new features from BootStrap and SASS.

The challenging part for this project was that my teammates lived in Australia. The time difference was something we had to focus on. Meetings were planned prior to the actual meetup, along with constant communication on the team’s discord channel.

Initab Clone

The Initab clone marked my first project into the Chingu program. Chingu is a flexible remote learning program for developers, and aspiring developers, who want to complete projects for the experience. This solo project was a prerequisite for a six week-long team project with a deadline of completion in fourteen days. To assist with the process, a discord channel was set up for anyone completing solo projects for Chingu. The Initab clone was based on a Chrome new tab extension for developers.

This project was coding in HTML, CSS and JavaScript. I used CSS position to mimic the same layout from the app. The requirements stated that the clone did not need all the features from the app, just the design, clock and date. I used JavaScript for the function of the clock. It keeps time and the date for any current time zone displayed. The challenge was the deadline. I had to make sure the time spent was focused on the requirements and not trying to overdo it.
